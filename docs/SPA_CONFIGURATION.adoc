== Overview

The SPA documentation generator is configured through three YAML files located in `config/spa/`:

* `ui_theme.yml` - Visual appearance (colors, typography, layout)
* `features.yml` - Feature toggles and behavior
* `templates.yml` - Template structure and components

Each configuration file can be customized or overridden using the `--config` option.

== Configuration files

=== ui_theme.yml

Controls the visual appearance of the documentation.

==== Structure

[source,yaml]
----
theme:
  colors:
    # Light mode colors
    ...

  dark_colors:
    # Dark mode colors
    ...

  typography:
    # Font settings
    ...

  layout:
    # Layout dimensions
    ...

  spacing:
    # Spacing units
    ...

  borders:
    # Border styles
    ...

  shadows:
    # Shadow effects
    ...

  transitions:
    # Animation timing
    ...
----

=== features.yml

Controls which features are enabled and their behavior.

==== Structure

[source,yaml]
----
features:
  search:
    enabled: true
    fuzzy_matching: true
    min_characters: 2
    debounce_ms: 300

  filtering:
    enabled: true
    show_count: true

  dark_mode:
    enabled: true
    default: "auto"
    persist: true

  navigation:
    enabled: true
    show_sidebar: true
    collapsible: true

  documentation:
    enabled: true
    show_annotations: true
----

=== templates.yml

Controls template structure and component arrangement.

==== Structure

[source,yaml]
----
templates:
  layout: "default"

  single_file:
    embed_resources: true
    inline_css: true
    inline_js: true

  multi_file:
    separate_resources: true
    directories:
      css: "css"
      js: "js"
      data: "data"

  components:
    - head
    - header
    - navigation
    - search
    - content
    - footer
    - scripts

  partials:
    schema_card:
      template: "components/schema_card"
    type_detail:
      template: "components/type_detail"
----

==== Layout options

[cols="1,2,1,2"]
|===
|Key |Purpose |Default |Description

|`layout`
|Layout template
|`"default"`
|Base layout to use
|===

==== Single-file options

[cols="1,2,1,2"]
|===
|Key |Purpose |Default |Description

|`embed_resources`
|Embed all resources
|`true`
|Include CSS/JS/data inline

|`inline_css`
|Inline CSS
|`true`
|Embed stylesheets

|`inline_js`
|Inline JavaScript
|`true`
|Embed scripts

|`minify`
|Minify resources
|`false`
|Minimize file size
|===

==== Multi-file options

[cols="1,2,1,2"]
|===
|Key |Purpose |Default |Description

|`separate_resources`
|Separate files
|`true`

|`directories.css`

== Custom configuration

=== Creating custom config

Create a custom configuration file combining all settings:

[source,yaml]
----
# my-spa-config.yml

# Theme customization
theme:
  colors:
    primary: "#10b981"
    secondary: "#6b7280"

  typography:
    font_family: "Inter, system-ui, sans-serif"
    font_size_base: "1.125rem"

# Feature customization
features:
  search:
    fuzzy_matching: true
    min_characters: 1

  dark_mode:
    default: "dark"

  navigation:
    show_sidebar: false

# Template customization
templates:
  layout: "default"
  single_file:
    minify: true
----

=== Using custom config

[source,bash]
----
lutaml-xsd generate-spa schemas.lxr \
  --config my-spa-config.yml \
  --output custom-docs.html
----

=== Partial configuration

You can provide partial configuration - only the specified values will override defaults:

[source,yaml]
----
# minimal-config.yml
theme:
  colors:
    primary: "#ec4899"  # Only change primary color
----

All other values will use defaults.

== Configuration examples

=== Minimal theme

Minimal, distraction-free documentation:

[source,yaml]
----
features:
  search:
    enabled: true
  filtering:
    enabled: false
  dark_mode:
    enabled: false
  navigation:
    show_sidebar: false

theme:
  colors:
    primary: "#000000"
    background_primary: "#ffffff"
  typography:
    font_family: "Georgia, serif"
----

[source,yaml]
----

  colors:

== Validation

=== Required fields

The following fields are required in configuration files:

*ui_theme.yml:*
- `theme.colors.primary`
- `theme.colors.background_primary`
- `theme.colors.text_primary`

*features.yml:*
- `features.search.enabled`
- `features.filtering.enabled`

*templates.yml:*
- `templates.layout`

=== Validation errors

Common validation errors:

==== Invalid color format

[source]
----
Error: Invalid color format for 'primary': 'blue'
Expected: Hex color (#RRGGBB) or CSS color name
----

Solution: Use hex format: `#0000ff`

==== Invalid dimension

[source]
----
Error: Invalid dimension for 'header_height': '64'
Expected: CSS unit (px, rem, em, etc.)
----

Solution: Include unit: `64px` or `4rem`

==== Unknown feature

[source]
----
Warning: Unknown feature 'my_feature' will be ignored
----

Solution: Check spelling or remove unknown features
